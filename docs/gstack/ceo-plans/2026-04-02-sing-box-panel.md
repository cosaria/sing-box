## Vision

### 10x Check

不是"Bash 脚本的 Go 重写"。是 **sing-box 生态的 3x-ui**。

3x-ui 用 ~41k stars 证明了代理面板有巨大需求。sing-box 生态目前只有 s-ui（~3k stars），完成度远不如 3x-ui。这是一个明确的机会。

10x 版本：单二进制同时是 sing-box 代理引擎 + HTTP API + TUI + Web UI 面板。零依赖部署。嵌入 sing-box 作为 Go 库实现热重载和内部监控。支持多用户、流量统计/限制、IP 限制、订阅管理、SSL 证书管理、Telegram Bot。

## Key Architecture Decisions

- **HTTP API 而非 Unix socket** — Phase 2 Web UI 直接复用同一 API
- **SQLite 而非 JSON 文件** — 为多用户/流量统计做准备
- **每个协议独立 typed Config struct** — 类型安全
- **DB 迁移系统从 Day 1** — Phase 2 加表无痛
- **API 认证从 Day 1** — Phase 2 远程访问时已就绪
- **不再依赖 jq** — Go 原生 encoding/json 处理所有 JSON 操作

## M0 Go/No-Go Criteria & Fallback

**Go 条件（必须全部满足）：**
1. `go get github.com/sagernet/sing-box` 能编译通过
2. `box.New()` 或 `libbox` 能启动 Shadowsocks inbound 并代理流量
3. 能通过 Go 接口获取 per-inbound 流量统计
4. 纯 Go 构建（无 CGO），支持交叉编译 linux/amd64 + linux/arm64

**如果 M0 失败：** 降级到方案 B（管理外部 sing-box 二进制）。API-First 架构不变，HTTP API + SQLite + TUI 全保留。只是 `internal/engine/` 改为进程管理而非库嵌入。流量统计改用 sing-box 的 Clash API 或日志解析。

## TUI-Server Process Model

TUI 和 API server 运行在同一个进程中：

```
sing-box (无参数)
  └─ main.go
       ├─ 启动 HTTP API server (goroutine)
       ├─ 启动 engine (goroutine)
       └─ 启动 Bubbletea TUI (主 goroutine, 阻塞)
           └─ TUI 通过 localhost HTTP 调用 API

sing-box serve
  └─ main.go
       ├─ 启动 HTTP API server (goroutine)
       ├─ 启动 engine (goroutine)
       └─ 阻塞等待信号 (SIGTERM/SIGINT)
```

TUI 退出时优雅关闭 API server 和 engine。`sing-box serve` 模式无 TUI，由 systemd 管理。

## Engine Crash Recovery

```
Engine goroutine:
  for {
      defer recover() → log panic, increment crash counter
      box = sing-box.New(options)
      box.Start()
      <-ctx.Done() // 等待关闭信号
      box.Close()
  }

如果 crash counter > 3 in 60s → 停止重试，通过 API 报告 "engine failed"
```

systemd `Restart=on-failure` 作为最后保障（整个进程崩溃时）。

## Token 类型说明

文档中有两种不同的 token：
1. **API auth token** — 认证管理 API 请求的 bearer token。存储在 SQLite `settings` 表。首次启动自动生成。
2. **Subscription token** — 订阅端点 `/sub/{token}` 中的 token。Phase 1 单用户，使用一个固定的随机 token。Phase 2 多用户时每个用户有独立 token。

## 流量统计实现

sing-box 通过 experimental Clash API 暴露流量数据。嵌入模式下：
- 在 sing-box 配置中启用 `experimental.clash_api`（仅内部使用，不对外暴露）
- Panel 的 stats collector 通过 Go 内部调用读取 Clash API 数据
- 每分钟采集一次，写入 SQLite `traffic_logs(inbound_tag, upload, download, timestamp)`
- Panel 的 HTTP API `/api/stats` 从 SQLite 查询并返回

注意：这意味着二进制内部有两个 HTTP listener。Clash API 绑定 127.0.0.1 的随机端口（或 Unix socket），仅供内部 stats collector 使用。Panel API 是面向用户的。

## Graceful Shutdown Sequence

```
1. 收到 SIGTERM/SIGINT（或 TUI 退出）
2. 停止接受新 API 请求（HTTP server shutdown with timeout）
3. 等待 in-flight API 请求完成（5s timeout）
4. 停止 stats collector
5. 关闭 sing-box engine（box.Close()，等待连接排空）
6. 关闭 SQLite 连接
7. 退出
```
