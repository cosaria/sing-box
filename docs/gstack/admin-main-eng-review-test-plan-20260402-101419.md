Generated by /plan-eng-review on 2026-04-02

Repo: sing-box (local)

## Test Strategy
- Framework: Go native `testing` + `go test`
- Every `internal/` package has a `_test.go`
- Engine and daemon use interfaces for mock-ability
- TUI: basic teatest smoke tests only

## Modules and Coverage Targets

### internal/engine/
- Start with valid config → engine running
- Start with invalid config → error returned, engine stays stopped
- Reload success → Close+New+Start, engine still running
- Reload failure → rolls back to Configured, error returned
- Stop → clean shutdown

### internal/config/
- List empty dir → empty slice
- List with configs → correct filenames
- Load valid Shadowsocks JSON → correct Config struct
- Load invalid JSON → error
- Add config → file written to CONF_DIR
- Delete existing config → file removed
- Delete nonexistent config → error

### internal/protocol/
- Shadowsocks.Fields() returns port, method, password specs
- Shadowsocks.JSON() → valid sing-box inbound JSON (validate with jq or json.Valid)
- Shadowsocks.URL() → correct SIP002 format (ss://base64@host:port#tag)
- Shadowsocks.Parse() → roundtrip: JSON() then Parse() recovers same Config
- Registry: register + lookup by name
- Registry: lookup unknown → error

### internal/platform/
- Detect amd64, arm64, unsupported arch
- Detect systemd, openrc, neither

### internal/service/
- Systemd unit file content matches expected template
- OpenRC init script content matches expected template

### internal/daemon/
- Server accepts JSON request, returns JSON response
- Client sends reload, receives success
- Client sends reload, receives error with message
- Client connects when no socket → daemon not running

### internal/share/
- Shadowsocks URL encoding (standard method)
- Shadowsocks URL encoding (SS2022 with base64 password)
- QR generation produces non-empty output

### internal/update/
- Parse GitHub API response → correct version string
- Atomic rename self-update (mock filesystem)

## Edge Cases
- Config file with invalid JSON in CONF_DIR
- Port already in use when starting engine
- Socket file left over from crashed daemon
- Empty CONF_DIR (no configs) → engine starts with no inbounds?

## Critical Paths
- Full lifecycle: add config → start engine → verify running → reload → verify still running → stop
- Daemon communication: management command → socket → daemon → reload → response back
