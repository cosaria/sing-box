# M3 Design Spec: TUI + Distribution

> Generated by brainstorming session on 2026-04-02

## Overview

M3 在 M1+M2 后端基础上，添加 Bubbletea TUI 交互界面、QR 码终端渲染、install.sh 安装脚本、自更新机制和 goreleaser 构建配置，交付 v0.1 可用版本。

## Design Decisions

| 决策点 | 选择 | 理由 |
|--------|------|------|
| TUI 模式 | 内嵌（TUI + server 同进程） | 一条命令搞定，零配置 |
| TUI 功能范围 | 完整覆盖 | 添加/查看/修改/删除 inbound + 统计 + 状态 |
| 分发方式 | GitHub Releases + goreleaser | 标准做法，一行命令安装 |
| 自更新 | 原子替换 | 简单可靠，YAGNI |
| QR 码 | 纯终端 Unicode 字符 | 零依赖，任何终端都能用 |

## Part 1: TUI Architecture

### 技术栈

- Bubbletea — TUI 框架（Model-Update-View pattern）
- Lipgloss — 终端样式（边框、颜色、对齐）
- go-qrcode — QR 码矩阵生成

### 文件结构

```
internal/tui/
├── tui.go              # App model, 顶层状态机, Run() 入口
├── menu.go             # 主菜单视图
├── add.go              # 添加 inbound 向导
├── list.go             # inbound 列表 + 详情/分享/QR/删除
├── edit.go             # 修改 inbound
├── status.go           # 引擎状态 + 订阅地址
├── stats.go            # 流量统计表格
├── client.go           # HTTP API client
├── qr.go               # QR 终端渲染
└── style.go            # Lipgloss 样式常量
```

### 状态机

```
MainMenu
  ├─ 添加配置  → AddWizard（选协议→输端口→确认→显示结果/QR→回主菜单）
  ├─ 查看配置  → InboundList → InboundDetail（分享链接/QR/编辑/删除）
  ├─ 修改配置  → InboundList → EditView（修改端口/settings→保存→回列表）
  ├─ 流量统计  → StatsView（汇总表格，r 刷新）
  ├─ 服务状态  → StatusView（engine 状态 + 订阅地址，r 刷新）
  └─ 退出
```

### TUI 与 Server 的进程模型

`sing-box`（无参数）：
1. 启动 HTTP API server（goroutine）
2. 启动 engine（goroutine）
3. 启动 stats collector（goroutine）
4. 启动 Bubbletea TUI（主 goroutine，阻塞）
5. TUI 通过 `client.go` 调 `http://127.0.0.1:{port}/api/*`
6. TUI 退出时优雅关闭 server + engine + collector

### API Client

```go
type Client struct {
    baseURL string
    token   string
    http    *http.Client
}

func NewClient(baseURL, token string) *Client
func (c *Client) Status() (*StatusResp, error)
func (c *Client) ListInbounds() ([]*Inbound, error)
func (c *Client) CreateInbound(protocol string, port uint16) (*Inbound, error)
func (c *Client) GetInbound(id int64) (*Inbound, error)
func (c *Client) UpdateInbound(id int64, port *uint16, settings *json.RawMessage) (*Inbound, error)
func (c *Client) DeleteInbound(id int64) error
func (c *Client) GetStats() ([]TrafficSummary, error)
func (c *Client) Reload() error
```

TUI 的每个视图只调 client 方法，不直接处理 HTTP。

## Part 2: TUI Interaction Details

### 主菜单

```
╭─ sing-box 管理面板 ─────────────────╮
│                                      │
│  > 添加配置                           │
│    查看配置                           │
│    修改配置                           │
│    流量统计                           │
│    服务状态                           │
│    退出                              │
│                                      │
╰──────────────────────────────────────╯
```

方向键上下选择，Enter 确认，q 退出。

### 添加配置向导（AddWizard）

4 步流程：
1. 选协议 → 列出已注册的 `protocol.All()`（Shadowsocks / VLESS-REALITY / Trojan）
2. 输端口 → 文本输入框，默认随机可用端口
3. 确认 → 显示摘要（协议+端口），Enter 创建
4. 创建成功 → 立即显示分享链接 + QR 码，按任意键回主菜单

### 查看配置（InboundList → Detail）

- 列表显示所有 inbound：`协议 | 端口 | Tag`
- 选中 Enter 进入详情：显示完整 settings、分享链接、QR 码
- 详情页快捷键：`d` 删除（二次确认 y/n）、`e` 编辑、`Esc` 返回列表

### 修改配置（EditView）

- 从查看配置详情页按 `e` 进入
- 可修改字段：端口、协议特定 settings
- 修改后自动调 API 更新 + reload engine
- 保存成功后返回详情页

### 流量统计（StatsView）

- 表格显示所有 inbound 累计上传/下载，人类可读格式（如 `52.4 MB`）
- `r` 手动刷新，`Esc` 返回主菜单

### 服务状态（StatusView）

- 显示：engine running/stopped、inbound 数量、API 地址、订阅地址
- `r` 手动刷新，`Esc` 返回主菜单

### QR 码渲染

使用 `github.com/skip2/go-qrcode` 生成 QR 矩阵，转换为 Unicode 半高块字符（`█`, `▀`, `▄`, ` `）在终端渲染。每行两个 QR 像素用一个字符表示，有效压缩高度。

## Part 3: install.sh

### 使用方式

```bash
bash <(curl -sL https://raw.githubusercontent.com/cosaria/sing-box/main/install.sh)
```

### 安装流程

1. 检测平台（OS + ARCH）：仅支持 linux/amd64 和 linux/arm64
2. 从 GitHub Releases 下载最新二进制（`sing-box-linux-{arch}`）
3. 安装到 `/usr/local/bin/sing-box`
4. 创建数据目录 `/usr/local/etc/sing-box/`
5. 运行 `sing-box service install` 安装系统服务
6. 启用并启动服务：`sing-box service start`
7. 等待服务启动后，打印 API token + 订阅地址

### 参数

- `-l` 本地安装（从当前目录 `go build`）
- `-v v0.1.0` 指定版本
- `-p http://proxy:port` 使用代理下载

## Part 4: Cobra 子命令

```
sing-box                    # 启动 TUI（内嵌 API server + engine）
sing-box serve              # 守护进程模式（已有）
sing-box update             # 自更新
sing-box service install    # 安装系统服务
sing-box service uninstall  # 卸载系统服务
sing-box version            # 显示版本号
```

### 自更新流程（`sing-box update`）

1. 请求 GitHub API `repos/cosaria/sing-box/releases/latest` 获取最新版本号
2. 对比编译时注入的 `main.Version`
3. 已是最新 → 提示并退出
4. 下载对应平台的新二进制到临时文件
5. 校验（文件大小 > 0，可执行权限）
6. 原子 `os.Rename` 替换当前二进制（`os.Executable()` 获取路径）
7. 打印：`更新完成: v0.1.0 → v0.2.0，请运行 sing-box service restart`

### service 子命令

复用 `internal/service/` 包：
- `install` → 调 `Manager.Install(binPath, dataDir)`
- `uninstall` → 调 `Manager.Uninstall()`

### version 子命令

打印编译时注入的版本号和 Go 版本：
```
sing-box panel v0.1.0 (go1.26.1 linux/amd64)
```

## Part 5: goreleaser

`.goreleaser.yml` 配置：
- 构建目标：linux/amd64 + linux/arm64
- CGO_ENABLED=0
- ldflags: `-X main.Version={{.Version}}`
- 二进制命名：`sing-box-{{.Os}}-{{.Arch}}`
- 发布到 GitHub Releases
- 生成 checksums.txt
