# M2 Design Spec: Protocol Framework + Traffic Stats + Subscription

> Generated by brainstorming session on 2026-04-02

## Overview

M2 在 M1 核心后端基础上，实现多协议支持、流量统计和订阅端点，将 sing-box panel 从"只能管 Shadowsocks"提升为"覆盖主流协议的完整面板"。

## Design Decisions

| 决策点 | 选择 | 理由 |
|--------|------|------|
| Protocol 抽象 | 接口 + registry | 统一 build、URL 生成、默认配置入口，不过度抽象 |
| 流量采集 | sing-box 内部 Go 接口 | 零额外开销，不需要内部 Clash API HTTP listener |
| 订阅认证 | 独立 token，无 Bearer | 客户端兼容性硬要求（Clash、v2rayN 等不支持自定义 Header） |
| 订阅格式 | Base64 分享链接列表 | 最大兼容性，YAGNI |
| VLESS 密钥 | 创建时自动生成 | 面板核心价值是自动化 |
| 密码生成 | 统一使用 UUID | 三协议凭据格式一致 |

## Part 1: Protocol Interface & Registry

### 文件结构

```
internal/protocol/
├── protocol.go          # Protocol 接口定义 + registry
├── shadowsocks.go       # Shadowsocks 实现
├── vless.go             # VLESS-REALITY 实现
└── trojan.go            # Trojan 实现
```

### Protocol 接口

```go
type Protocol interface {
    Name() string                                           // "shadowsocks", "vless", "trojan"
    DisplayName() string                                    // "Shadowsocks", "VLESS-REALITY", "Trojan"
    DefaultSettings(port uint16) (string, error)            // 自动生成默认配置 JSON
    BuildInbound(ib *store.Inbound) (option.Inbound, error) // 构建 sing-box inbound option
    GenerateURL(ib *store.Inbound, host string) string      // 生成分享链接
}
```

### Registry

```go
var registry = map[string]Protocol{}

func Register(p Protocol)           // 注册协议
func Get(name string) Protocol      // 按名称获取，不存在返回 nil
func All() []Protocol               // 列出全部已注册协议
```

各协议在 `init()` 中调用 `Register()` 自注册。

### 变更影响

- `engine/build.go` — 删除 switch-case 和 `shadowsocksSettings` struct，改为 `protocol.Get(ib.Protocol).BuildInbound(ib)`
- `api/inbound.go` — 创建 inbound 时：如果 settings 为空，调用 `protocol.DefaultSettings(port)` 自动生成；如果有 settings，与默认值 merge
- Tag 生成统一为 `{protocol}-{port}`

## Part 2: Protocol Implementations

### Shadowsocks

**Settings JSON:**
```json
{
  "method": "2022-blake3-aes-128-gcm",
  "password": "<UUID>"
}
```

- `DefaultSettings()`: 生成 UUID 作为 password，默认 method 为 `2022-blake3-aes-128-gcm`（SS2022 128 位，UUID 是 16 字节匹配 128 位 key）
- `BuildInbound()`: 映射到 `option.ShadowsocksInboundOptions`
- `GenerateURL()`: `ss://base64(method:password)@host:port#tag`

### VLESS-REALITY

**Settings JSON:**
```json
{
  "uuid": "<UUID>",
  "flow": "xtls-rprx-vision",
  "reality": {
    "private_key": "<X25519 private key>",
    "public_key": "<X25519 public key>",
    "short_id": "<8 字符 hex>",
    "handshake": {
      "server": "www.microsoft.com",
      "server_port": 443
    }
  }
}
```

- `DefaultSettings()`: 自动生成 UUID、X25519 密钥对、8 字符 hex short_id。默认 handshake 目标为 `www.microsoft.com:443`
- `BuildInbound()`: 映射到 `option.VLESSInboundOptions` + Reality TLS 配置
- `GenerateURL()`: `vless://uuid@host:port?type=tcp&security=reality&sni=server&fp=chrome&pbk=public_key&sid=short_id&flow=xtls-rprx-vision#tag`

### Trojan

**Settings JSON:**
```json
{
  "password": "<UUID>"
}
```

- `DefaultSettings()`: 生成 UUID 作为 password
- `BuildInbound()`: 映射到 `option.TrojanInboundOptions`（Phase 1 无 TLS，适合前置 Nginx/Caddy 或本地测试）
- `GenerateURL()`: `trojan://password@host:port#tag`

### API 创建流程

`POST /api/inbounds` 的 `settings` 字段变为可选：
1. 无 settings → 调用 `protocol.DefaultSettings(port)` 全自动生成
2. 有 settings → 与默认配置 merge（用户字段覆盖默认值）

## Part 3: Traffic Stats

### 新组件

```
internal/stats/
└── collector.go    # 定期采集 + SQLite 写入
```

### 数据库 Migration 2

```sql
CREATE TABLE traffic_logs (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    inbound_tag TEXT NOT NULL,
    upload INTEGER NOT NULL,
    download INTEGER NOT NULL,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP
)
```

### 采集机制

1. `Engine` 新增 `GetTrafficCounters() map[string]TrafficCounter` 方法，从 `box.Box` 实例的 router/inbound 获取 per-inbound upload/download 字节数
2. `Collector` goroutine 每 60 秒调用一次，计算增量（当前值 - 上次快照），写入 `traffic_logs` 表
3. Engine reload 时计数器重置 → Collector 检测到值减小时，将当前值视为新的增量

### API 端点

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/stats` | 所有 inbound 累计流量 |
| GET | `/api/stats/{tag}` | 单个 inbound 流量历史 |

**响应示例 (`/api/stats`):**
```json
[
  {"tag": "ss-8388", "upload": 1048576, "download": 52428800},
  {"tag": "vless-443", "upload": 2097152, "download": 104857600}
]
```

## Part 4: Subscription Endpoint

### 路由

`GET /sub/{token}` — 公开路由，不经过 tokenAuth 中间件。

### Token 管理

- 首次启动时自动生成，存入 `settings` 表 key = `sub_token`（与 `api_token` 独立）
- 启动时终端打印订阅 URL：`http://{listen_addr}/sub/{token}`

### 输出流程

1. 从 store 列出所有 inbound
2. 对每个 inbound 调用 `protocol.Get(ib.Protocol).GenerateURL(ib, host)`
3. 所有 URL 用 `\n` 拼接
4. 整体 Base64 编码
5. 响应 Content-Type: `text/plain; charset=utf-8`

### Host 获取

从 HTTP 请求的 `Host` header 获取服务器公网 IP/域名，无需额外配置。

### 新文件

`internal/api/subscription.go`

### server.go 路由变更

```go
// 公开路由（无 auth）
r.Get("/sub/{token}", s.handleSubscription)

// 认证路由
r.Group(func(r chi.Router) {
    r.Use(tokenAuth(token))
    // ... 现有 API 路由
})
```

## Part 5: Change Summary

### 新增文件

| 文件 | 职责 |
|------|------|
| `internal/protocol/protocol.go` | Protocol 接口 + registry |
| `internal/protocol/shadowsocks.go` | Shadowsocks 实现 |
| `internal/protocol/vless.go` | VLESS-REALITY 实现 |
| `internal/protocol/trojan.go` | Trojan 实现 |
| `internal/stats/collector.go` | 流量采集器 |
| `internal/api/subscription.go` | 订阅端点 handler |

### 修改文件

| 文件 | 变更 |
|------|------|
| `internal/engine/build.go` | 删除 switch-case，改用 protocol registry 分发 |
| `internal/engine/engine.go` | 新增 `GetTrafficCounters()` 方法 |
| `internal/store/store.go` | 新增 migration 2（traffic_logs 表） |
| `internal/api/server.go` | 路由分组（公开 vs 认证）、注册 stats 和 subscription 路由 |
| `internal/api/inbound.go` | 创建时调用 protocol.DefaultSettings()、merge 逻辑 |
| `cmd/sing-box/main.go` | 初始化 Collector、生成 sub_token、打印订阅 URL |

### 不变的文件

- `internal/store/inbound.go` — Inbound 模型不需要改
- `internal/platform/` — 不涉及
- `internal/service/` — 不涉及
- `internal/api/middleware.go` — 认证逻辑不变

### 依赖变更

无需新增第三方依赖。sing-box v1.13.5 的 go.mod 已间接包含 VLESS/Reality 相关包。
